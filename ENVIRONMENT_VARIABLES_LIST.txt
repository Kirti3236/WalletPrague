GROUP 2: DATABASE (Supabase)
-----------------------------

OPTION A: Use Connection String (RECOMMENDED - Bypasses DNS Issues)
--------------------------------------------------------------------
NAME: DATABASE_URL
VALUE: postgresql://postgres:[PASSWORD]@db.klnbdurpeojayizubunc.supabase.co:6543/postgres
(Get this from Supabase Dashboard ‚Üí Settings ‚Üí Database ‚Üí Connection string)

If you use DATABASE_URL, you DON'T need to set DB_HOST, DB_PORT, DB_USERNAME, DB_PASSWORD, DB_NAME separately.

OR

OPTION B: Use Separate Variables
----------------------------------
NAME: DB_HOST
VALUE: [Your Supabase Host]
Example: db.klnbdurpeojayizubunc.supabase.co
(Just hostname, no https:// or port)

NAME: DB_PORT
VALUE: 6543
(Use 6543 for connection pooler OR 5432 for direct connection)

NAME: DB_USERNAME
VALUE: postgres
(Usually "postgres" for Supabase)

NAME: DB_PASSWORD
VALUE: [Your Supabase Database Password]
(This is the password you created when setting up Supabase project)

NAME: DB_NAME
VALUE: postgres
(Usually "postgres" for Supabase)

REQUIRED FOR BOTH OPTIONS:
NAME: DB_SYNC
VALUE: true
(Set to "false" in production if you don't want auto-sync)

NAME: DB_LOGGING
VALUE: false
(Set to "true" if you want to see SQL queries in logs)

GROUP 3: JWT & SECURITY (Generate These!)
------------------------------------------
‚ö†Ô∏è  IMPORTANT: Generate these secrets first using terminal:

Run these commands to generate secrets:
openssl rand -base64 32    # For JWT_SECRET
openssl rand -base64 32    # For JWT_REFRESH_SECRET
openssl rand -base64 32    # For SESSION_SECRET
openssl rand -base64 32    # For ENCRYPTION_KEY
openssl rand -base64 16    # For ENCRYPTION_IV

Then add:

NAME: JWT_SECRET
VALUE: [Generated secret - 32+ characters]

NAME: JWT_EXPIRES_IN
VALUE: 15m

NAME: JWT_REFRESH_SECRET
VALUE: [Generated secret - 32+ characters]

NAME: JWT_REFRESH_EXPIRES_IN
VALUE: 7d

NAME: SESSION_SECRET
VALUE: [Generated secret - 32+ characters]

NAME: ENCRYPTION_KEY
VALUE: [Generated secret - exactly 32 characters]

NAME: ENCRYPTION_IV
VALUE: [Generated secret - exactly 16 characters]

NAME: BCRYPT_ROUNDS
VALUE: 12

================================================================================
TROUBLESHOOTING
================================================================================

If variable not working:
- Check spelling (case-sensitive!)
- Verify no extra spaces
- Check value is correct
- Restart service after adding variables

If build fails:
- Check all REQUIRED variables are added
- Verify JWT secrets are long enough (32+ chars)
- Check database credentials are correct
