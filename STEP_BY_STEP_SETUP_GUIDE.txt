STEP 4: SET UP EMAIL SERVICE
================================================================================

TIME: 5-10 minutes

You have TWO options. Choose ONE:

OPTION A: GMAIL SMTP (Easiest - Use Client's Gmail)
----------------------------------------------------

4A.1 Access Client's Gmail Account
   - Go to: https://mail.google.com
   - Sign in with client's email and password

4A.2 Enable 2-Step Verification (REQUIRED)
   - Go to: https://myaccount.google.com

     * Click "2-Step Verification"
     * Follow setup wizard
     * You'll need access to client's phone for verification code

4A.3 Create App Password
   - Go to: https://myaccount.google.com/apppasswords

     * App: Mail
     * Device: Other (Custom name)
     * Name: "YaPague API" or "YaPague Backend"
   - Click "Generate"
   - A 16-character password will appear (e.g., "abcd efgh ijkl mnop")
   - COPY THIS IMMEDIATELY (you won't see it again!)

4A.4 Save Credentials
   ‚òê Gmail address: _____________________________
   ‚òê Gmail password: _____________________________ (original password)
   ‚òê Gmail App Password: _____________________________ (16-char, remove spaces when using)

================================================================================
TROUBLESHOOTING
================================================================================

PROBLEM: Can't verify email
SOLUTION: 
- Check spam folder
- Request new verification email
- Make sure you have access to client's email inbox

PROBLEM: Can't create Gmail App Password
SOLUTION:
- 2-Step Verification MUST be enabled first
- Go to Google Account ‚Üí Security ‚Üí Enable 2-Step Verification
- Then try creating App Password again

PROBLEM: Database password not saved
SOLUTION:
- Supabase: Go to Settings ‚Üí Database ‚Üí Reset database password
- Save the new password immediately!

PROBLEM: API Key/Secret not saved
SOLUTION:
- Cloudinary: Can regenerate API Secret (Settings ‚Üí Security)
- SendGrid: Must create new API Key (old one can't be retrieved)

PROBLEM: Can't access client's email
SOLUTION:
- Ask client to provide email access
- Or ask client to verify accounts themselves
- Or use a shared email account
