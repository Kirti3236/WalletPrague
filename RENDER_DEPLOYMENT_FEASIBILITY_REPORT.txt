5. DEPLOYMENT STEPS (GUIDE)
================================================================================

5.1 Step 1: Set Up FREE PostgreSQL Database
-------------------------------------------
RECOMMENDED: Use Supabase or Neon (100% FREE, no expiration)

Option A: Supabase (Recommended - Easiest Setup)
1. Go to https://supabase.com
2. Sign up for free account
3. Create new project
4. Wait for database to provision (2-3 minutes)
5. Go to Settings ‚Üí Database
6. Copy "Connection string" (URI format)
7. Note down:
   - Connection String (postgresql://...)
   - Or extract: Host, Port (5432), Database, User, Password
   - Password is shown once - save it!

Option B: Neon (Alternative - More Storage)
1. Go to https://neon.tech
2. Sign up for free account
3. Create new project
4. Copy connection string from dashboard
5. Note down connection details

Option C: Render PostgreSQL (NOT RECOMMENDED - Expires after 90 days)
1. Go to Render Dashboard
2. Click "New +" ‚Üí "PostgreSQL"
3. Choose "Free" tier (90 days free, then $7/month)
4. Note down connection details

5.2 Step 2: Create Web Service
-------------------------------
1. Go to Render Dashboard
2. Click "New +" ‚Üí "Web Service"
3. Connect your Git repository (GitHub/GitLab/Bitbucket)
4. Configure:
   - Name: yapague-backend (or your choice)
   - Region: Choose closest to your users
   - Branch: main/master
   - Root Directory: (leave empty if root)
   - Environment: Node
   - Build Command: npm install && npm run build
   - Start Command: node dist/main.js
   - Plan: Free

5.3 Step 3: Configure Environment Variables
--------------------------------------------
In Web Service settings ‚Üí Environment:
Add all required environment variables (see section 4.2)

Important:
- DB_HOST: Use INTERNAL database host (for free tier)
- DB_PORT: 5432
- DB_USERNAME: From PostgreSQL service
- DB_PASSWORD: From PostgreSQL service
- DB_NAME: From PostgreSQL service

5.4 Step 4: Set Up FREE File Storage (REQUIRED)

RECOMMENDED: Cloudinary (Easiest, 25GB free forever)

  2. Sign up for free account

  4. Copy:

     - API Secret
  5. Add to environment variables:

  6. Add to environment variables:

     - SMTP_USER=your-email@gmail.com

  6. Copy API key
  7. Add to environment variables:
     - SENDGRID_API_KEY=your-api-key
     - SENDGRID_FROM_EMAIL=your-verified-email@domain.com
  8. Install: npm install @sendgrid/mail
  9. Update code to use SendGrid
  Limit: 100 emails/day (enough for dev/testing)

Option C: Mailgun (Alternative)
  1. Go to https://www.mailgun.com
  2. Sign up for free account
  3. Verify domain or use sandbox domain
  4. Get API key from dashboard
  5. Add to environment variables
  6. Install: npm install mailgun.js

  Limit: 5,000 emails/month (best free limit)
