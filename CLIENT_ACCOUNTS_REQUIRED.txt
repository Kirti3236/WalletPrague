2. SUPABASE - PostgreSQL Database (REQUIRED)
================================================================================

PURPOSE: Store all your application data (users, transactions, etc.)

SIGN UP LINK: https://supabase.com

STEPS FOR CLIENT:
1. Go to https://supabase.com
2. Click "Start your project" or "Sign Up"
3. Sign up with GitHub (recommended) or Email
4. Verify email
5. Click "New Project"
6. Fill in:
   - Project Name: yapague-db (or any name)
   - Database Password: (create strong password - SAVE THIS!)
   - Region: Choose closest to your location
7. Click "Create new project"
8. Wait 2-3 minutes for database to be created

WHAT CLIENT NEEDS TO PROVIDE:
- Supabase project URL
- Database password (the one they created)
- Connection details (we'll guide them to find this)

HOW TO GET CONNECTION DETAILS (Guide for Client):
1. In Supabase dashboard, go to "Settings" (gear icon)
2. Click "Database"
3. Scroll to "Connection string"
4. Copy the connection string (postgresql://...)
5. OR provide these separately:
   - Host
   - Port (usually 5432)
   - Database name
   - Username
   - Password

NOTES:
- FREE forever (500MB storage)
- No credit card required
- No expiration date
- This is the chosen database service (Supabase only)

NOTES:

- No credit card required

4. EMAIL SERVICE - Choose ONE Option
================================================================================

PURPOSE: Send emails from your API (notifications, password reset, etc.)

OPTION A: GMAIL SMTP (Easiest - Use Existing Gmail)
----------------------------------------------------
REQUIREMENTS: Client needs a Gmail account

STEPS FOR CLIENT:
1. Client should use their existing Gmail account
   OR create new Gmail: https://accounts.google.com/signup
2. Go to Google Account: https://myaccount.google.com
3. Click "Security" (left sidebar)
4. Enable "2-Step Verification" (if not already enabled)
5. After enabling, go back to Security
6. Scroll down to "App passwords"
7. Click "App passwords"
8. Select:
   - App: Mail
   - Device: Other (Custom name)
   - Name: "YaPague API"
9. Click "Generate"
10. Copy the 16-character password (e.g., "abcd efgh ijkl mnop")
    - Remove spaces when using: "abcdefghijklmnop"

WHAT CLIENT NEEDS TO PROVIDE:
- Gmail address (e.g., "client@gmail.com")
- Gmail App Password (the 16-character password generated)

NOTES:
- FREE (uses existing Gmail)
- 500 emails/day limit (enough for dev/testing)
- Very easy to set up

OPTION B: SENDGRID (More Reliable - Recommended)
------------------------------------------------
SIGN UP LINK: https://sendgrid.com

STEPS FOR CLIENT:
1. Go to https://sendgrid.com
2. Click "Start for Free" or "Sign Up"
3. Fill in:
   - Email address
   - Password
   - Company name (optional)
4. Verify email address
5. Complete signup form
6. Verify your email (check inbox)

WHAT CLIENT NEEDS TO PROVIDE:
After signup and email verification:
1. Go to Settings → API Keys
2. Click "Create API Key"
3. Name: "YaPague API"
4. Permissions: Select "Full Access" or "Mail Send"
5. Click "Create & View"
6. Copy the API Key (starts with "SG.")
7. Send API key securely

NOTES:
- FREE forever (100 emails/day)
- No credit card required
- More reliable than Gmail SMTP
- Better for production later

RECOMMENDATION: Ask client to choose ONE
- Gmail: If they already have Gmail (easiest)
- SendGrid: If they want more reliable service (recommended)
